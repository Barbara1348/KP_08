7) Как браузер использует закэшированные
изображения?

Браузер отправляет запрос на сервер (GET /images/photo.jpg)
Сервер отправляет изображение с кэш-заголовками (Cache-Control, ETag)
При повторном запросе браузер проверяет кэш и заголовки. Если ресурс актуален,
изображение берётся из кэша

8) как происходит Кэширование на стороне сервера и CDN
Пользователь делает запрос на ресурс (изображение, JS, CSS)
Сервер проверяет, есть ли закэшированная версия ресурса
Если ресурс кэширован, он передаётся быстрее (Cache-Control)
CDN доставляет ресурс с ближайшего сервера для пользователя
Если ресурс обновился, CDN кэширует новую версию

42)Зачем нужна асинхронность ? 

асинхронность на стороне клиента предотвращает «зависание» интерфейса при загрузке данных, сохраняя сайт отзывчивым для пользователя. На стороне сервера она позволяет эффективно обрабатывать тысячи одновременных соединений, не блокируя ресурсы в ожидании ответа от баз данных или сторонних сервисов.

43)Как работает хук useState в React ?

Хук useState принимает начальное значение и возвращает массив из двух элементов: текущего состояния и функции для его обновления. Когда вызывается функция обновления, React сохраняет новое значение и запускает повторный рендеринг компонента, чтобы интерфейс отобразил актуальные данные.

44)Как работает хук useEffect в React  ?

Хук useEffect позволяет выполнять побочные эффекты (запросы к API, подписки или манипуляции с DOM) после того, как компонент отрисовался на экране. Он принимает функцию и массив зависимостей, запуская код повторно только при изменении указанных значений или один раз при монтировании, если массив пуст.

45)Отличие функциональных компонентов от классовых ?

Функциональные компоненты — это обычные функции, которые используют хуки для управления состоянием и логикой, в то время как классовые компоненты основаны на классах ES6 и методах жизненного цикла. Функциональный подход является современным стандартом React, так как он делает код более лаконичным и избавляет от сложностей, связанных с использованием контекста this.

46)Как работает хук useMemo в React ?

Хук useMemo кэширует результат вычисления и пересчитывает его только в том случае, если изменилась одна из указанных зависимостей. Это позволяет избежать тяжелых повторных вычислений при каждом рендере, оптимизируя производительность приложения.

47)Как работает хук useCallback в React ?

Хук useCallback возвращает мемоизированную версию функции, которая сохраняется между рендерами и обновляется только при изменении значений в массиве зависимостей. Это необходимо для оптимизации производительности, чтобы предотвратить лишние перерисовки дочерних компонентов, которые полагаются на стабильность ссылок при передаче функций через пропсы.

48)Что такое Promise ?

Promise — это объект, представляющий результат асинхронной операции, который может находиться в одном из трёх состояний: ожидание (pending), исполнено (fulfilled) или отклонено (rejected). Он позволяет удобно обрабатывать завершение задачи с помощью цепочек методов .then() и .catch(), избавляя код от глубокой вложенности колбэков.




49)Как работает  цикл for в js ?

Цикл for последовательно выполняет блок кода, пока заданное условие остается истинным, используя три части: инициализацию, условие и финальное выражение (шаг). Сначала инициализируется переменная-счетчик, затем перед каждой итерацией проверяется условие, и в случае его истинности выполняется тело цикла, после чего происходит обновление счетчика.

50)Как работает  цикл while в js ?

Цикл while многократно выполняет блок кода до тех пор, пока заданное условие остается истинным. Проверка условия происходит перед каждой итерацией, поэтому, если оно изначально ложно, тело цикла не выполнится ни разу.

51)Как работает метод перебора массива map в js ?

Метод map создаёт новый массив, заполняя его результатами вызова функции для каждого элемента исходного массива по порядку. Он позволяет трансформировать данные, не изменяя (не мутируя) оригинальный массив, что делает его основным инструментом для отображения списков данных в интерфейс.

52)Как работает метод перебора массива forEach в js ?

Метод forEach последовательно выполняет переданную функцию-колбэк для каждого элемента массива по одному разу. Он всегда возвращает undefined и используется для совершения действий над данными (побочных эффектов), а не для трансформации массива в новую структуру

53)Что такое dto ? 

DTO (Data Transfer Object) — это объект, предназначенный для передачи данных между различными частями приложения или между сервером и клиентом. Он не содержит бизнес-логики и служит для группировки только необходимых полей в одну структуру, помогая отделить внутренние модели данных от внешнего интерфейса.

54)Что такое tailwind и зачем он нужен ?

Tailwind CSS — это утилитарный CSS-фреймворк, который позволяет стилизовать элементы, комбинируя готовые низкоуровневые классы (например, flex, pt-4, text-center) прямо в HTML или JSX.

Он нужен для значительного ускорения верстки и упрощения поддержки проекта, так как избавляет от необходимости писать огромные CSS-файлы и придумывать названия для каждого класса.

55) Что такое express ( node.js )


Express.js — это минималистичный и гибкий фреймворк для Node.js, который упрощает разработку серверной части веб-приложений и API. Он предоставляет удобные инструменты для управления маршрутизацией (routing), обработки HTTP-запросов и внедрения промежуточных обработчиков (middleware), избавляя разработчика от написания сложного низкоуровневого кода на чистом Node.js.

56)Что такое SQLite 

SQLite — это компактная встраиваемая система управления базами данных, которая не требует отдельного сервера и хранит всю информацию в одном обычном файле на диске. Она работает как библиотека, подключаемая напрямую к приложению, что делает её идеальным выбором для мобильных устройств, браузеров и небольших программ, где важна простота и скорость работы.

57)Что такое props ?

Props — это входные данные, которые передаются от родительского компонента дочернему для настройки его внешнего вида или поведения. Они доступны только для чтения и позволяют создавать универсальные переиспользуемые компоненты, которые отображают разные данные в зависимости от полученных значений.




58)Что такое виртуальный DOM ?

Виртуальный DOM — это легкая копия реального DOM, хранящаяся в памяти в виде дерева JavaScript-объектов. Она позволяет React сначала просчитывать изменения в памяти, а затем точечно обновлять только нужные элементы на странице, что значительно повышает производительность интерфейса.

59)Отличие виртуального DOM от обычного ?

Обычный DOM — это интерфейс браузера, в котором любое изменение данных требует пересчета стилей и разметки для всего дерева, что замедляет работу приложения. 

Виртуальный DOM — это копия в памяти, которая позволяет сначала найти минимальную разницу между старым и новым состоянием (diffing) и точечно обновить только изменившиеся элементы, значительно экономя ресурсы.
